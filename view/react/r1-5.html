Section 1: React Introduction

ReactDOM.render()

Syntax

ReactDOM.render(WhatToRender, WhereToRender);

Explanation

As I mentioned in the last lesson ReadDOM is necessary to view (or render) our app on the screen. This is largely because of the render() function that is part of ReactDOM. Simply put, if we want something to show in the screen using React we need to "render" it. Let's walk through an example.

Practice

  1. Go back to our index.js file from earlier. You should see one line that says.

    ReactDOM.render(<App />, document.getElementById('root'));

    This line says to render a Component named "App" to the webpage where it finds the id "root".

  2. Comment out this line. Then copy/paste or type in the following in the line below it.

    ReactDOM.render('hello', document.getElementById('root'));

    You should see your local page refresh automatically, if not refresh it manually. If everything worked you should see a blank page with the word 'hello' on the top left of the window.

  3. Let's take it a step further and render a JavaScript variable. I'll adjust my code as follows:

    var sum = 5+9+31;
    ReactDOM.render(sum, document.getElementById('root'));

    Refresh your page browser if necessary and you should see a '45' on the top left of my browser window.

  4. Last, but not least, let's render an HTML element.

    const e = React.createElement('strong', null, 'Hello There' );
    ReactDOM.render(e, document.getElementById( 'root' ));

    SURPRISE! Bet you didn't expect that one. Since we're in a javascript file we can't simply type in <strong> or any HTML tag. This forces us to use a React createElement function to render an HTML element. More about the createElement() function can be found on reactjs.org

Before moving on to the quiz, I suggest taking some time to play with the ReactDOM.render() function. We won't worry too much about React.createElement() at this point, so long as you are aware that we cannot display HTML tags without it.
