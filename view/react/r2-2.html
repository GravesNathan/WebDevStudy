JSX Overview

JSX Elements, Attributes, and Rules

We'll cover a lot of JSX groundwork in this lesson so be sure to take your time and pay attention.

JSX Elements

JSX Elements are the most basic piece of JSX. Generally they look exactly, or almost exactly, like their HTML element counterparts. Check out this example <h1> JSX element example.

<h1>Hello world</h1>

See any difference between this h1 JSX element and an h1 element in HTML? No...? Well good, because you shouldn't. In fact the only noticible difference between JSX elements and their corresponding HTML elements is JSX Elements are found in JavaScript files!

You should also know that JSX Elements can be used anywhere that traditional JavaScript can be used. For example, we can save JSX Elements to variables or constants in JavaScript (shown below).

var navBar = <nav>I am a nav bar</nav>;

JSX Attributes

JSX Attributes consist of name, value pairs in the format name="value" used in the opening tag of the JSX Element. This is essentially the same as HTML attributes. Here are some examples.

<a href="example.com"><h1>Go To Example.com</h1></a>;

var myName = < strong > My name is myname! < strong />;

Rules for using JSX

When using JSX there are a few rules that distinguish it from HTML that you should be familiar with.

  1. Nesting JSX:
    JSX elements can be wrapped in each other just just like in HTML.

    <a href="example.com"><h1>Go To Example.com</h1></a>;

    Please note that the above two examples are both actually on 1 line each. It might appear to be multiple lines depending on your device but there are not any actual line breaks. This single line pieces of code can be hard to follow, but otherwise works.

  2. Multi-Line JSX:
    In order to use multiple lines for a JSX expression we need to encapsulate it in parinthesis.

    (
       <a href="example.com">
         <h1>
          Go To Example.com
        </h1>
      </a>
    );

  3. Only ONE outer JSX Element:
    When using multiple lines of JSX in an expression the outer-most element must be the starting and ending tags. For example this is not allowed.

    (
    <h1>Main Header</h1>
    <h2>Sub-Header 1</h2>
    <p>Stuff goes here</p>
    <h2>Sub-Header 2</h2>
    )

    If we simply wrap the elements in a single outer element then we're good.

    (
    <div>
      <h1>Main Header</h1>
      <h2>Sub-Header 1</h2>
      <p>Stuff goes here</p>
      <h2>Sub-Header 2</h2>
    </div>
    )

Summary

  1. JSX Elements are the most basic piece of JSX code and work a lot like their HTML counterparts.
  2. JSX attributes are delcared in the same way as HTML attributes.
  3. JSX elements can be nested inside each other just like HTML elements.
  4. JSX expressions that take up more than one line need to be encompased in parinthesis.
  5. The starting and ending tags of JSX expressions must be the same JSX Element.
